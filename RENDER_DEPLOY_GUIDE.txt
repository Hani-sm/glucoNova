STEP 3: CREATE POSTGRESQL DATABASE
================================================================================

1. From Render Dashboard, click "New +"

2. Select "PostgreSQL"

3. Configure:
   Name: gluconova-db
   Database: gluconova
   User: gluconova
   Region: Choose closest to you (e.g., Oregon USA)
   PostgreSQL Version: 16
   Plan: Free

4. Click "Create Database"

5. IMPORTANT: Copy the "Internal Database URL"
   Format: postgresql://gluconova:password@dpg-xxx/gluconova
   
   Save this - you'll need it in Step 5!

6. Wait for database status to show "Available" (1-2 minutes)

STEP 4: CREATE WEB SERVICE
================================================================================

1. From Render Dashboard, click "New +"

2. Select "Web Service"

3. Click "Connect a repository"

4. Find and select your "gluconova" repository

5. Configure the service:
   
   Name: gluconova
   
   Region: Same as database (important for performance)
   
   Branch: main
   
   Root Directory: (leave blank)
   
   Runtime: Node
   
   Build Command:
   npm install && npm run build
   
   Start Command:
   npm start
   
   Plan: Free

6. Click "Advanced" to expand advanced settings

7. Add Auto-Deploy: Yes (already default with GitHub)

================================================================================
STEP 5: CONFIGURE ENVIRONMENT VARIABLES
================================================================================

In the "Environment Variables" section, add these:

1. DATABASE_URL
   Value: (Paste the Internal Database URL from Step 3)
   Example: postgresql://gluconova:password@dpg-xxx/gluconova

2. NODE_ENV
   Value: production

3. JWT_SECRET
   Value: Generate a secure secret by running locally:
   node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
   
   Then paste the output here (should be 64 characters)

4. PORT
   Value: 8080
   (Render automatically sets this, but explicit is better)

================================================================================
TROUBLESHOOTING
================================================================================

Issue: Build fails
Solution:
- Check build logs for specific error
- Verify package.json scripts are correct
- Ensure Node version â‰¥ 18 (set in package.json engines)

Issue: Database connection error
Solution:
- Verify DATABASE_URL is correct (check for typos)
- Ensure database status is "Available"
- Check database and web service are in same region

Issue: App starts but crashes immediately
Solution:
- Check logs: Dashboard â†’ Logs
- Verify all environment variables are set
- Ensure JWT_SECRET is set
- Run database migrations: npm run db:push

Issue: 503 Service Unavailable
Solution:
- Free tier apps sleep after 15 min of inactivity
- First request takes 30-60 seconds to wake up
- This is normal for free tier
- Upgrade to paid tier ($7/mo) for always-on

Issue: WebSocket connection fails
Solution:
- Render fully supports WebSockets
- Check browser console for errors
- Verify wss:// protocol is used in production
